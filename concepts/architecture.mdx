## End-to-end flow

1. **User intent**: user selects a claim to prove (e.g. “over 18”, “human”).
2. **Private fetch (zkTLS)**: extension fetches authentic Web2 data without revealing it to the website or verifier.
3. **Local proving (zkVM)**: extension runs SP1/RISC0 (compiled to WASM) to compute the claim and generate a proof.
4. **Aggregation + verification (zkVerify)**: proofs are aggregated and verified quickly on zkVerify.
5. **Attestation**: zkVerify emits/records an **on-chain attestation**.
6. **Consumption**: websites query attestations through a simple API and gate access without seeing the private inputs.

## Proof systems

zkVerify is designed to work with multiple proof types, including (examples):

- Groth16
- FFLONK
- Boojum
- zkVM proofs (SP1 / RISC0)
