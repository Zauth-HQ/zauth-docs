## Frontend (extension UI)

- React (for extension UI and onboarding)
- Wallet / DID linking as needed for attestation identity

## Proving

- **zkTLS**: TLSNotary-style workflows for private Web2 data
- **zkVM**: SP1 or RISC Zero (Rust → WASM)

## Verification

- Submit and poll attestations via a thin client layer (e.g. `zkVerifyJS` / API client)
- Prefer aggregation where possible for cost efficiency

## Backend (optional)

- Aggregation service for multi-event proofs
- Settlement adapters for chain-agnostic attestations (L1/L2)
